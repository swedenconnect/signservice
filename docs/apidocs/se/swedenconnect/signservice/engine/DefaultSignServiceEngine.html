    • resumeAuthentication

      @Nonnull protected SignServiceProcessingResult resumeAuthentication(@Nonnull HttpUserRequest httpRequest, @Nonnull EngineContext context) throws UnrecoverableSignServiceException
      Is called when the engine is invoked after the user has been directed to the authentication service. When the authentication is resumed it means that the issued authentication credentials (assertion) is being processed by the authentication handler. If everything is ok, the control is the passed back to the "finalize" phase.
      Parameters:
      httpRequest - the HTTP request
      context - the engine context
      Returns:
      a SignServiceProcessingResult object
      Throws:

    • completeAuthentication

      protected void completeAuthentication(@Nonnull HttpUserRequest httpRequest, @Nonnull AuthenticationResult authnResult, @Nonnull EngineContext context) throws UnrecoverableSignServiceException, SignServiceErrorException
      The "complete authentication" method is invoked after the authentication handler has reported a successful user authentication. At this point we know that the handler has asserted that the required user attributes were presented during the authentication, but we still need to check some additional things. This includes asserting that the signature message was displayed (if required) and making sure that we received attributes from the authentication needed to create the certificate contents.
      Parameters:
      httpRequest - the HTTP request
      authnResult - the authentication result
      context - the engine context
      Throws:
      UnrecoverableSignServiceException - for unrecoverable errors
      SignServiceErrorException - for errors that should be passed back to the client (as an error response)
